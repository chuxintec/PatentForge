# 运行编排验证样例系统 V1.0

## 1. 目标

本样例用于验证 PatentForge 的 OpenCode 编排流程是否能够自动完成“代码生成 → 审核 → 结果落盘”的闭环。

## 2. 功能范围

系统包含以下简单功能：

1. 读取输入任务配置
2. 校验任务名称、执行人、优先级
3. 生成任务执行摘要
4. 将结果输出为文本报告
5. 记录基础运行日志

## 3. 输入输出

- 输入：任务名称、执行人、优先级、截止日期
- 输出：标准化任务摘要、校验结果、文本报告内容

## 4. 模块说明

### 4.1 配置读取模块
- 接收原始任务对象
- 填充默认值

### 4.2 校验模块
- 校验任务名称是否为空
- 校验优先级是否在 low / medium / high 范围内
- 校验截止日期格式

### 4.3 摘要生成模块
- 组合任务基本信息
- 输出适合归档的摘要文本

### 4.4 日志模块
- 记录开始、校验、输出三个阶段

## 5. 实现要求

1. 使用 TypeScript 单文件实现
2. 代码中应有清晰的数据流：输入 -> 校验 -> 处理 -> 输出
3. 需要包含基础错误处理
4. 需要能够直接运行并打印结果
